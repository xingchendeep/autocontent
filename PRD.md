# 产品需求文档（PRD）

## 1. 文档信息

| 项目 | 内容 |
|------|------|
| 产品名称 | AutoContent Pro |
| 产品定位 | 面向内容创作者的“视频内容一键改写为多平台文案”工具 |
| 当前版本 | v0.4（修订版） |
| 修订日期 | 2026-03-13 |
| 文档状态 | 可进入开发与验证阶段 |

## 2. 产品概述

### 2.1 一句话介绍
输入一条视频链接或视频脚本，1 分钟内生成适配 10 个平台的发布文案，帮助创作者显著缩短内容分发时间。

### 2.2 背景与机会
- 多平台运营已成为内容创作者的常态，但不同平台对标题长度、语气、结构、标签和互动方式要求不同。
- 当前创作者通常需要重复改写同一内容，单条视频分发文案准备时间约 3-5 小时。
- 市面上已有通用 AI 写作工具，但缺少围绕“视频转多平台文案”的垂直产品，尤其缺少中文场景下的批量分发能力。

### 2.3 产品愿景
成为自媒体创作者和小型内容团队的标准分发工作台，让“1 份内容，多平台分发”变成默认工作流。

### 2.4 核心价值
- 节省时间：从 3-5 小时缩短至 1 分钟以内
- 降低成本：减少对外包文案与人工改写的依赖
- 提升一致性：保持统一内容基调，同时适配不同平台风格
- 提高发布效率：帮助用户更快完成内容上线和测试

## 3. 目标与范围

### 3.1 业务目标

#### MVP（上线后 1 个月）
- 获得 20-30 名种子用户
- 至少 5 名愿意持续使用的高频用户
- 生成成功率 >= 95%
- 平均生成时长 <= 30 秒
- 至少完成 5 次真实用户访谈与复盘

#### v1.0（上线后 3 个月）
- 付费转化率 >= 3%
- MRR >= $500
- 月留存 >= 25%
- 用户满意度 >= 4.0/5

#### v2.0（上线后 6 个月）
- 付费用户 >= 100
- MRR >= $3,000
- 月流失率 <= 8%
- NPS >= 40

### 3.2 产品目标
- 让用户无需登录即可体验核心生成能力
- 支持主流中文内容平台的文案生成
- 提供可直接复制使用的结构化结果
- 建立可追踪、可优化的模板与数据反馈闭环

### 3.3 非目标（当前阶段不做）
- 不直接替代视频剪辑、字幕编辑、封面设计工具
- 不在 MVP 阶段支持自动发布到各平台
- 不在 MVP 阶段支持团队协作、审批流和企业权限管理
- 不在 MVP 阶段支持多语种国际化运营

## 4. 目标用户

| 用户类型 | 核心特征 | 主要诉求 | 价格敏感度 |
|------|------|------|------|
| 个人创作者 | 1-10 万粉，多平台运营 | 快速生成可用文案 | 中 |
| 小型工作室 | 3-10 人，代运营多个账号 | 提升团队产能与稳定性 | 中低 |
| MCN / 品牌内容团队 | 管理 10+ 账号 | 标准化分发流程 | 低 |

### 4.1 典型用户画像

#### 用户画像 A：个人知识博主
- 每周产出 2-4 条长视频
- 同时运营抖音、小红书、B 站、公众号
- 痛点是写文案耗时，且平台风格把握不稳定

#### 用户画像 B：代运营工作室成员
- 每天要给多个客户分发内容
- 需要批量、稳定、可复制的文案输出
- 更关心效率、历史记录、多人复用模板

---

## 5. 用户痛点与需求

### 5.1 现有痛点
- 同一视频要手动改写 5-10 份版本
- 每个平台内容格式与互动话术差异大
- 文案质量依赖个人经验，不稳定
- 发布前准备过久，错过最佳发布时间

### 5.2 用户核心任务
- 输入视频链接或脚本
- 选择需要分发的平台
- 获取各平台可直接使用的文案
- 针对单个平台快速重生成或微调
- 保存历史结果，便于复用

---

## 6. 解决方案

### 6.1 核心方案
围绕“视频内容 -> 多平台文案”建立结构化工作流：
1. 获取内容原始输入（视频链接 / 手动文本）
2. 识别内容主题、受众和关键信息
3. 按平台模板并行生成文案
4. 以卡片形式展示结果并支持复制、重生成、保存

### 6.2 产品核心能力
- 平台模板化：每个平台有独立生成规则
- 批量生成：一次性输出多个平台版本
- 风格适配：标题、正文、标签、语气自动适配
- 低门槛体验：无需登录即可试用 MVP 核心功能

---

## 7. 用户流程

```text
输入内容 -> 选择平台 -> 点击生成 -> 查看结果 -> 复制使用 / 单平台重生成 -> 保存历史
```

### 7.1 关键体验要求
- 首次使用 30 秒内理解产品价值
- 从输入到看到结果不超过 3 个主要操作
- 失败时给出明确原因与下一步建议

## 9. 用户故事与验收

### 9.1 用户故事
- 作为个人创作者，我希望粘贴一段视频脚本后快速得到多个平台文案，这样我可以当天完成分发。
- 作为代运营人员，我希望一次生成多个平台版本并保留历史记录，这样我可以批量服务客户。
- 作为付费用户，我希望在生成失败时只重试失败的平台，这样我不需要重新等待全部内容。

### 9.2 顶层验收标准
- 用户从进入首页到复制任一平台文案，3 步内完成
- 生成结果中，至少 80% 可直接轻改后用于发布
- 失败时提供“失败原因 + 可执行建议 + 重试入口”

## 10. 商业模式

### 10.1 定价建议
| 套餐 | 价格 | 主要权益 | 目标用户 |
|------|------|---------|---------|
| 免费版 | $0 | 限部分平台、标准速度、基础历史记录 | 体验用户 |
| 创作者版 | $29/月 | 全平台、快速生成、更多历史记录 | 个人创作者 |
| 工作室版 | $79/月 | 优先队列、更多配额、团队预留能力 | 小团队 |
| 企业版 | $199/月 | API / 专属支持 / 定制能力 | 品牌与机构 |

### 10.2 当前建议优化
- 不建议免费版“完全无限制”，容易导致滥用和成本失控
- 建议免费版限制平台数量、速度或高级功能，而不是只靠频率限制
- 价格差异应体现于“效率、历史、模板、协作、服务”，不只是调用额度

### 10.3 退款与支持
- 首次订阅支持 7 天退款
- 超过 7 天按使用情况酌情处理
- 提供邮箱支持，v1.0 增加站内聊天

## 12. 技术方案

### 12.1 MVP 技术栈
- 前端：Next.js 14 + React + Tailwind CSS
- 后端：Next.js API Routes
- 模型：通义千问 Plus（可预留多模型切换）
- 部署：Vercel
- 数据：MVP 用 localStorage 保存本地历史
- 监控：Vercel Analytics + 基础日志

### 12.2 v1.0 升级
- Supabase：用户、历史记录、认证
- Lemon Squeezy：支付与订阅
- Upstash：计数与限流
- Resend：通知邮件

### 12.3 v2.0 升级
- Cloudflare R2：文件存储
- QStash：异步任务
- 多模型路由：Claude / GPT / 通义
- 审核与安全服务：敏感词过滤、风控策略

### 12.4 技术原则
- 优先保障生成成功率和响应速度
- 模板与模型解耦，便于迭代
- 所有成本可追踪，便于后续优化单次调用成本

## 13. 非功能需求

### 13.1 性能
- 首页首屏加载 <= 3 秒
- 平均生成时长 <= 30 秒
- 支持至少 10 名用户同时使用 MVP
- 可用性 >= 99%

### 13.2 安全
- API Key 仅保存在服务端
- 接口具备限流和异常监控
- 敏感数据全程 HTTPS 传输
- 登录态具备过期与刷新机制

### 13.3 兼容性
- 浏览器：Chrome、Edge、Safari、Firefox 主流版本
- 设备：桌面优先，移动端可用
- 语言：简体中文优先，英文放到 v1.0 后评估

### 13.4 可维护性
- 关键路径具备基础测试
- 核心接口有日志与错误码
- 模板规则可配置，不硬编码在页面层

## 18. 待确认问题

- 免费版是否允许无限次生成，还是限制每日次数 / 平台数量 / 速度
- 链接提取能力是否仅覆盖 YouTube 与 B 站，还是后续扩展到抖音、小红书等
- 10 个平台是否都在 MVP 中上线，还是分批上线以降低模板维护成本
- 当前模型是否只用通义，还是默认预留备用模型方案
- 企业版是否真的需要在早期提供 API，以免分散资源

## 19. 结论

该项目方向明确，场景具体，具备较强的 MVP 验证价值。建议优先验证三件事：
- 用户是否真的愿意高频使用“多平台一键改写”
- 模板输出质量是否足够支撑复制使用
- 免费到付费的价值差是否足够明显

如果这三点成立，AutoContent Pro 有机会成为面向中文内容创作者的高频分发效率工具。
